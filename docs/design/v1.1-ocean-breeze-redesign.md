# Design V1.1 — "Ocean Breeze" redesign

Status: **Approved** (plan only — not yet implemented). Supersedes the V1 "Sea &
Sand" presentation layer. V1 (issues #2–#11) is tagged `v1.0`.

## Why

V1 shipped every page, the enquiry flow, SEO/a11y, and the logo — but the visual
language leans generic ("too AI-ish") and **card-heavy**: nearly every section is
a row of identical shadcn cards. V1.1 refreshes the palette and replaces the
repeated card grids with varied, cinematic components, without touching the data
layer, CMS collections (except adding Testimonials), enquiry flow, or SEO.

Sources: the color palette and the marquee/hero/testimonial/bento components are
drawn from the 21st.dev catalog (see ids below); three were the user's own
bookmarks (⭐).

## Palette — Ocean Breeze

Adopted from the 21st.dev "Ocean Breeze" theme, mapped onto our existing token
system (`globals.css` HSL vars + `tailwind.config` named colors). AA-safe
variants are derived where the raw theme colors fail WCAG contrast (same
technique as the V1 `coral-ink` fix).

| Token | New value | Replaces | Note |
|---|---|---|---|
| Page background | `#f0f8ff` airy alice-blue | cream `#fbf7f0` | the defining shift |
| Card / surface | `#ffffff` | white | unchanged |
| Ink / text | `#243447` deep slate | ink `#12333f` | strong contrast |
| Primary (brand/links) | `#0e7490` ocean teal | sea `#0c6e8a` | continuity with V1 |
| CTA / action | `#15803d` fresh green | coral-ink `#bf4420` | AA-safe from theme `#22c55e` |
| Soft band / secondary | `#e0f2fe` sky | sand | section backgrounds, chips |
| Accent / mint | `#d1fae5` | sky-tint | subtle highlights, hovers |
| Border | `#d8e6f0` cool hairline | warm border | — |

- Type: keep Fraunces (display) + Inter (body). (Ocean Breeze ships DM Sans/Lora
  — optional future switch.)
- The background flip re-opens every contrast pairing tuned in #9 (SEO/a11y), so
  the axe suite must pass again across all key pages.

## Testimonials — new CMS collection

A Payload `Testimonials` collection so staff manage real quotes:

- `quote` (textarea, required), `authorName` (text, required),
  `authorLocation` (text), `trip` (text/relationship, optional),
  `rating` (number 1–5, optional), `avatar` (upload, optional),
  `featured` (checkbox).
- Migration + a few seeded entries.
- `listTestimonials` / `listFeaturedTestimonials` in the server data layer (the
  established test seam), with an integration test. Circular Testimonials renders
  from these view models.

## Rollout

One `feat/redesign` branch, built **section by section**: token foundation first,
then each section end-to-end (build → drive in preview → axe + smoke → review),
on top of the current V1 stack. Full suite + `/code-review` before the final push.

## Out of scope

Data layer, existing CMS collections (except adding Testimonials), the enquiry
flow, SEO/sitemap/OG, and the signature mark/logo. Payment is still never taken
(ADR-0002). No new locales (ADR-0003 unchanged).

## Risks

- **Perf:** the scroll-expansion hero is heavier — re-check LCP; keep
  reduced-motion holds throughout ("graceful, not static", ADR-0004).
- **A11y:** cool background changes every contrast pairing — re-run axe.
- **Warmth:** Ocean Breeze is cool; warm destination photography must still read
  well against it (it does — photos pop against a cool frame).
- **Paid retrievals:** ~8 21st Pro component fetches; some may paywall.
